误解与争执:h5到底是什么？

发表于 2017-05-25 | 分类于 javascript | 阅读次数

起源

　　HTML5大概是大部分人听到h5所联想到的第一个名词。
　　HTML的全称是：HyperText Mark-up Language(超文本标记语言),而HTML5则是它的第五次重大修改。当我们说到h5应用的时候，很多人脑海中可能会浮现基于HTML5开发的混合app(Hybrid APP,微信小程序)或web应用(微信公众号，一些手机端web应用)，国内对于h5这个还有针对性，那就是仅限于移动端。
　　我们都知道手机端浏览器比较新，大多使用webkit内核，与桌面ie11相同，支持es5乃至更高规范，故而我们开发混合app的时候肯定可以使用HTML5，CSS3，甚至es6。大多数项目经理应该都是这样想的，所以在跟客户解释技术规范的时候会说：我们将使用HTML5、CSS3…..balabala(一大堆懒得念,干脆简化成h5更顺口)。似乎没什么问题，大家也能接受，然后口口相传，h5这个名词也就第一次有了它的定义：“移动端的一套包含HTML5在内的技术规范”。

第一次分化

　　正当大家为了证明自己git到了新的热点，而不断在朋友圈刷新这个名词的时候,一大部分围观群众下意识的认为h5就是HTML5，然后固执己见，这就产生了第一次分歧。
　　大家你一言我一语都认为自己才是真正的理解了h5，认为h5就是HTM5的这些人正是代表了某些从不涉足技术领域的产品经理形象，他们的理解也有明显的错误——光靠HTML5是无法开发APP的。
　　接触过原生android开发的人都知道，原生应用是使用xml来构建界面的，到了“h5应用”这里则是换成了html5，细心的朋友可能已经发现了，html不过是xml基于W3C规范的一种延伸，xml作为一种标示型语言，仅仅只能用来构建界面上的一些标记以及展示内容文本，假如h5=html5,那么试想一下“html5应用”将会是一个布满黑色文字和蓝色链接的网页，而且仅仅只是网页。是的，没有使用cordova根本不能称之为app。
　　h5=html5?假如你也有过类似的想法，趁现在赶紧纠正还来得及！

第二次分化

　　在项目经理和产品经理唇枪舌剑的讨论h5到底是不是html5的时候，事情终于到了开发者这里。然后产生了更有趣的变化。
　　开始实施的开发者发现，手机端的环境并不如之前所想象的那么统一，在兼容性的问题上，这里的混乱并不输给PC WEB端。让我们看看实际的开发环境：
　　安卓经过了几次版本的更换，不同厂商的定制，ios6前后也使用了不同内核，这些都是我们在开发hybrid app和微信公众号的时候极其头疼的问题。而当我们开发手机端web应用的时候事情变得更加失去控制，各大浏览器(UC,腾讯,百度,360,搜狗……)他们有的使用webkit内核，有的使用trident，有的是gecko,甚至有的是以上的变种…..苦不堪言的开发者最终只能妥协以兼容性为首要考虑，决定放弃使用HTML5,CSS3,es6等最新的技术规范，而改用html2,css,es5。是的，以h5命名的app其实很多都不包含html5！这些从我们用到的众多开源技术可以瞥见：angularjs2被称为最适合移动端开发的语言，却还是把es6编译成es5运行。
　　那么问题来了，且不说h5=html5等于无稽之谈，就算是h5=“移动端的一套包含HTML5在内的技术规范”。也根本不成立！
　　h5这个名词尽管如今还被人们津津乐道，但实际从产生到流产并没有经历太多的时间。
　　可项目还要继续，h5这个词还是口口相传，只是在一部分人这里又变了意义：“让你的app可以拖拉拽划且兼容性好的一套技术，不管是否包含html5”

总结

　　如今h5在不同的场景代表了不同的意义，当我们口头讨论技术规范的时候，h5回归了它本来的意义:“移动端的一套包含HTML5在内的技术规范”,而当我们着手开发的时候h5则代表了“让你的app可以拖拉拽划且兼容性好的一套技术，不管是否包含html5”
　　至于你说h5=html5？还是快打消这个念头吧(笑)

angularjs应用超简单启动

发表于 2017-05-17 | 分类于 javascript | 阅读次数

为什么写

在前端技术一片混沌的现在，angularjs作为出身名门的开源框架，其优点有:
先进的双向绑定(这里婊一下浙江大学那三个研究生导师),
完整的面向对象，模块化分层，多向路由，单例等诸多优点。深受企业用户的青睐。
相信大家已经想试试了，在早期的angularjs1.0时代，我们通过手动引入依赖，手动搭建结构来完成前端框架的构造。
如今angularjs已经更新到4.0版本，并且从2.0开始原生支持es6规范，我们需要更多组件使他能兼容es5以及早期版本，
这使得框架更加的复杂。
本篇文章就是教大家怎么用几秒钟的时间快速搭建angularjs helloWord。

工具

1.git
2.nodejs
工具的用法大家都懂，相信没有必要赘述。

步骤

1.打开npm
(如果没有切换源的，这里最好配置一下:
npm更换到淘宝镜像: npm install cnpm -g --registry=https://registry.npm.taobao.org
)
2.定位到你的工作空间
3.打开git，执行git clone https://github.com/angular/quickstart.git hello2ng2
4.执行 npm install(早期版本由gulp构建，如今angularjs已经4.0了，也省略了许多步骤)
5.npm start


浏览器会自动启动并看到以上页面

可以看到整个结构都很清晰了，想要尝鲜的玩家现在可以大展身手啦！
